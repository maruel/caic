# Authentication Plan: Login with GitHub / GitLab

## Overview

Add OAuth 2.0 login via GitHub and GitLab to caic. When enabled, every API
endpoint requires a JWT session. Multiple users can each have their own tasks
and preferences. Existing deployments using `GITHUB_TOKEN`/`GITLAB_TOKEN`
environment variables continue to work unchanged when the OAuth configuration
is absent.

This document is the single source of truth for the implementation. Execute
the phases in order; sections marked **parallel** may be handed to independent
subagents simultaneously.

---

## 1. Operator Prerequisites

Before the feature can be used, the operator must:

1. **Register OAuth applications** on GitHub and/or GitLab:
   - GitHub: Settings → Developer settings → OAuth Apps → New OAuth App
     - Authorization callback URL: `{CAIC_EXTERNAL_URL}/api/v1/auth/github/callback`
     - Scopes granted by the user flow: `repo`, `read:user`
   - GitLab: User Settings → Applications → Add new application
     - Redirect URI: `{CAIC_EXTERNAL_URL}/api/v1/auth/gitlab/callback`
     - Scopes: `api`, `read_user`

2. **Set a public domain name** (`CAIC_EXTERNAL_URL`) reachable by the
   browser and by the OAuth provider. This is a hard requirement; OAuth
   callbacks cannot use `localhost`.

3. **Set a session secret** (`CAIC_SESSION_SECRET`): a random hex string of
   at least 32 bytes (e.g. `openssl rand -hex 32`).

Auth is **disabled by default**. The server starts in legacy unauthenticated
mode if `CAIC_EXTERNAL_URL` or `CAIC_SESSION_SECRET` is absent, or if no
OAuth provider is configured. All existing behaviour is preserved in that mode.

## 3. Data Model

### `~/.config/caic/users.json`

Stored with the same atomic-write pattern as `preferences.json`.

```json
{
  "version": 1,
  "users": [
    {
      "id": "usr_<ksid>",
      "provider": "github",
      "providerID": "12345678",
      "username": "alice",
      "avatarURL": "https://avatars.githubusercontent.com/u/12345678",
      "accessToken": "gho_...",
      "refreshToken": "",
      "tokenExpiry": "0001-01-01T00:00:00Z",
      "createdAt": "2026-01-01T00:00:00Z",
      "lastSeenAt": "2026-01-01T00:00:00Z"
    }
  ]
}
```

Fields:
- `id`: generated by `ksid.NewID()` with a `"usr_"` prefix on first insert
- `provider`: `"github"` or `"gitlab"` — matches `forge.Kind`
- `providerID`: the provider's numeric user ID as a string (stable, unlike username)
- `accessToken`: OAuth access token; used for forge API calls on behalf of the user
- `refreshToken`: empty for GitHub (no refresh tokens issued); non-empty for GitLab
- `tokenExpiry`: zero value when no expiry

### JWT payload

HS256, signed with `CAIC_SESSION_SECRET`, 30-day TTL.

```json
{
  "uid": "usr_<ksid>",
  "prv": "github",
  "usr": "alice",
  "iat": 1234567890,
  "exp": 1237159890
}
```

Implemented entirely with stdlib: `crypto/hmac`, `crypto/sha256`,
`encoding/base64`, `encoding/json`. No new Go dependency.

JWT structure: `base64url(header) + "." + base64url(payload) + "." + base64url(sig)`
where `sig = HMAC-SHA256(secret, base64url(header) + "." + base64url(payload))`.

### Web session

The JWT is delivered to the browser via a `Set-Cookie` header on the OAuth
callback redirect. Cookie attributes:
- Name: `caic_session`
- `HttpOnly` — not accessible to JavaScript
- `SameSite=Strict` — no CSRF risk, no need for extra CSRF tokens
- `Secure` — set only when `CAIC_EXTERNAL_URL` starts with `https://`
- `Max-Age`: 30 days (matches JWT TTL)
- `Path=/`

Using a cookie (rather than localStorage) means `EventSource` connections
carry the session automatically — no changes needed to the SSE code path.

### Android session

The JWT is delivered via a redirect to the custom scheme `caic://auth?token=<jwt>`.
Android stores it in `EncryptedSharedPreferences` and injects it as
`Authorization: Bearer <token>` into every OkHttp request.

### OAuth state (CSRF protection)

On `/auth/{provider}/start`, generate a 16-byte random hex `state` string.
Sign it: `state + "." + hex(HMAC-SHA256(secret, state))`. Set as an
`HttpOnly SameSite=Strict` cookie named `caic_oauth_state` with `Max-Age=600`
(10 minutes). On callback, read the cookie, split on `"."`, re-compute HMAC,
verify. Reject if missing or invalid. This is entirely stateless — no
server-side storage is needed for the OAuth handshake.

### Per-user preferences

Each user's preferences live at `~/.config/caic/preferences/<userID>.json`
(e.g. `preferences/usr_abc123.json`). In no-auth mode, the server uses
`preferences/default.json` with a one-time migration from the old
`preferences.json` path on startup.

The `preferences.Store` type does not change. The server holds a
`sync.Map` of `userID → *preferences.Store` and lazily opens/creates stores
on first access per user. A helper `prefsForUser(ctx) (*preferences.Store, error)`
reads the user from context and returns (or creates) their store.

## 4. New API Endpoints

All auth endpoints are exempt from the auth middleware.

| Method | Path | Auth required | Description |
|---|---|---|---|
| `GET` | `/api/v1/auth/github/start` | No | Redirect to GitHub OAuth |
| `GET` | `/api/v1/auth/github/callback` | No | Handle GitHub callback, issue cookie, redirect |
| `GET` | `/api/v1/auth/gitlab/start` | No | Redirect to GitLab OAuth |
| `GET` | `/api/v1/auth/gitlab/callback` | No | Handle GitLab callback, issue cookie, redirect |
| `GET` | `/api/v1/auth/me` | Yes | Return current user as `UserResp` |
| `POST` | `/api/v1/auth/logout` | Yes | Clear `caic_session` cookie, return 200 |

`start` handlers accept an optional `?return=app` query parameter. When
present, the post-callback redirect target is `caic://auth?token=<jwt>` instead
of the default web redirect to `/`. The allowed values of `return` are
hardcoded to `""` (web) and `"app"` (Android) — no open-redirect risk.

### New DTO types (add to `dto/v1/types.go`)

Register `getMe` and `logout` in the route table (`v1.Routes`) so they are

### Phase 1 — `backend/internal/auth` package (sequential foundation)

> No subagent parallelism here; each file builds on the previous.

Create the package `backend/internal/auth/`. All files in this package are
new; none touch existing code.

#### `auth/types.go`

```go
// Package auth implements JWT session management and OAuth 2.0 login
// for GitHub and GitLab.
package auth

import (
    "time"
    "github.com/caic-xyz/caic/backend/internal/forge"
)

// User represents an authenticated caic user.
type User struct {
    ID           string     // "usr_<ksid>"
    Provider     forge.Kind // "github" | "gitlab"
    ProviderID   string     // provider's numeric user ID as string
    Username     string
    AvatarURL    string
    AccessToken  string     // OAuth access token for forge API calls
    RefreshToken string     // empty for GitHub; may be set for GitLab
    TokenExpiry  time.Time  // zero value means no expiry
    CreatedAt    time.Time
    LastSeenAt   time.Time
}

// Claims are the fields embedded in the JWT payload.
type Claims struct {
    UserID   string
    Provider forge.Kind
    Username string
    IssuedAt time.Time
    Expiry   time.Time
}
```

#### `auth/store.go`

Mirrors `preferences.Store` exactly: `sync.Mutex`, in-memory cache of the
full file, atomic write via rename. File version field for future migrations.

Public API:
```go
type Store struct { /* unexported */ }

// Open reads or creates users.json at path.
func Open(path string) (*Store, error)

// UpsertUser creates or updates a user matched by (Provider, ProviderID).
// On create: generates a new "usr_<ksid>" ID and sets CreatedAt.
// On update: updates tokens, AvatarURL, Username, LastSeenAt.
// Returns the upserted User.
func (s *Store) UpsertUser(u User) (User, error)

// FindByProviderID returns the user with the given provider+ID pair, or false.
func (s *Store) FindByProviderID(provider forge.Kind, providerID string) (User, bool)

// FindByID returns the user with the given internal ID, or false.
func (s *Store) FindByID(id string) (User, bool)
```

The on-disk format has a `version: 1` field. Use a private `usersFile`
struct for JSON marshal/unmarshal to keep `User` clean of JSON tags.

#### `auth/session.go`

Implement HS256 JWT with stdlib only — no new dependency.

```go
// IssueToken creates a signed JWT for the user. ttl is typically 30 days.
func IssueToken(u User, secret []byte, ttl time.Duration) (string, error)

// ValidateToken parses and verifies a JWT. Returns Claims or an error.
func ValidateToken(token string, secret []byte) (*Claims, error)
```

Implementation notes:
- Header: `{"alg":"HS256","typ":"JWT"}` — base64url-encode, never change
- Payload fields: `uid`, `prv`, `usr`, `iat`, `exp` (Unix seconds)
- Signature: `HMAC-SHA256(secret, encodedHeader + "." + encodedPayload)`
- Validation: check signature, check `exp > time.Now().Unix()`

#### `auth/state.go`

CSRF state cookie helpers for the OAuth flow.

```go
// GenerateState returns a 16-byte random hex string.
func GenerateState() (string, error)

// SignState returns "state.hex(HMAC-SHA256(secret, state))".
func SignState(state string, secret []byte) string

// ValidateState splits cookie value on ".", re-computes HMAC, returns
// the bare state string. Returns ("", false) on any mismatch.
func ValidateState(cookie string, secret []byte) (string, bool)

// StateCookieName is "caic_oauth_state".
const StateCookieName = "caic_oauth_state"
```

#### `auth/oauth.go`

Provider-agnostic OAuth 2.0 Authorization Code exchange. Uses `net/http` only
— no `golang.org/x/oauth2` dependency.

```go
// ProviderConfig holds the OAuth 2.0 endpoint configuration for one provider.
type ProviderConfig struct {
    ClientID     string
    ClientSecret string
    AuthURL      string   // e.g. "https://github.com/login/oauth/authorize"
    TokenURL     string   // e.g. "https://github.com/login/oauth/access_token"
    UserInfoURL  string   // e.g. "https://api.github.com/user"
    Scopes       []string
    RedirectURI  string   // "{CAIC_EXTERNAL_URL}/api/v1/auth/{provider}/callback"
}

// GitHubConfig returns a ProviderConfig for github.com.
// Scopes: ["repo", "read:user"].
func GitHubConfig(clientID, secret, externalURL string) ProviderConfig

// GitLabConfig returns a ProviderConfig for a GitLab instance.
// Scopes: ["api", "read_user"].
// gitlabURL defaults to "https://gitlab.com".
func GitLabConfig(clientID, secret, gitlabURL, externalURL string) ProviderConfig

// AuthURL returns the provider's authorization URL with the state param.
func (c ProviderConfig) AuthURL(state string) string

// ExchangeCode exchanges an authorization code for tokens.
// Returns accessToken, refreshToken (may be empty), expiry (may be zero).
func ExchangeCode(ctx context.Context, cfg ProviderConfig, code string) (access, refresh string, expiry time.Time, err error)

// FetchUserInfo fetches the user's identity from the provider.
// Returns providerID (string form of numeric ID), username, avatarURL.
func FetchUserInfo(ctx context.Context, cfg ProviderConfig, accessToken string) (providerID, username, avatarURL string, err error)
```

Implementation notes for `ExchangeCode`:
- POST to `TokenURL` with `application/x-www-form-urlencoded` body:
  `grant_type=authorization_code&code=...&redirect_uri=...&client_id=...&client_secret=...`
- Set `Accept: application/json` on the request
- GitHub returns `{"access_token":"...","token_type":"bearer"}` (no expiry, no refresh)
- GitLab returns `{"access_token":"...","refresh_token":"...","expires_in":7200}`

Implementation notes for `FetchUserInfo`:
- GET `UserInfoURL` with `Authorization: Bearer <accessToken>`
- GitHub response: `{"id": 12345678, "login": "alice", "avatar_url": "..."}`
- GitLab response: `{"id": 12345678, "username": "alice", "avatar_url": "..."}`
- `providerID` must be the string form of the numeric `id` field (not the login/username, which can change)
- GitHub: if user has a private email, `email` may be empty; ignore email entirely — use `login` as username

#### `auth/middleware.go`

```go
type contextKey struct{}

// Middleware validates the caic_session cookie (or Authorization: Bearer header
// as a fallback for Android/API clients). Injects *User into context.
// When secret is nil (auth disabled), passes through unconditionally.
func Middleware(store *Store, secret []byte) func(http.Handler) http.Handler

// RequireUser returns a 401 JSON error if no authenticated user is in context.
// Used to protect all non-auth API routes when auth is enabled.
func RequireUser(next http.Handler) http.Handler

// UserFromContext returns the authenticated user, or (nil, false) in no-auth mode.
func UserFromContext(ctx context.Context) (*User, bool)
```

Token lookup order in `Middleware`:
1. `caic_session` cookie (web)
2. `Authorization: Bearer <token>` header (Android, curl)

On validation failure: do **not** return 401 from the middleware itself —
just leave the context without a user. `RequireUser` is the gatekeeper.
This allows the config endpoint to remain unprotected while other routes
are gated.

#### `server.go` — task ownership

In `createTask`: read user from context; set `task.OwnerID = user.ID` (or

`task.OwnerID == user.ID`.

In per-task handlers (`terminateTask`, `sendInput`, `syncTask`, `getTaskDiff`,
`getTaskCI`, etc.): after looking up the task, if auth is enabled and
`task.OwnerID != "" && task.OwnerID != user.ID`, return 403.

#### New file `backend/internal/server/auth.go`

HTTP handlers for the six auth endpoints. Implement:

- **`startOAuth(provider)`**: generate state, set `caic_oauth_state` cookie,
  redirect to provider's auth URL. Check that the provider is configured;
  return 404 if not. Accept `?return=app` query param; store it as a prefix
  in the state value so the callback knows the redirect target:
  `state = "web:" + randomState` or `state = "app:" + randomState`.
  The full cookie value is `SignState(state, secret)`.

- **`callbackOAuth(provider)`**: validate state cookie (reject if missing or
  invalid HMAC; clear the cookie either way), exchange code for tokens, fetch
  user info, upsert user in store, issue JWT, set `caic_session` cookie.
  Extract the `"web:"`/`"app:"` prefix from the validated state to decide
  redirect:
  - web: `http.Redirect(w, r, "/", http.StatusFound)` — cookie carries auth
  - app: `http.Redirect(w, r, "caic://auth?token="+url.QueryEscape(jwt), http.StatusFound)`

- **`getMe`**: return `UserResp` from context user.

- **`logout`**: set `caic_session` cookie with `Max-Age=-1` to clear it;
  return `StatusResp{Status: "ok"}`.

#### `main.go` additions

- Read new env vars into `server.Config`
- Validate: if `CAIC_EXTERNAL_URL` is set but `CAIC_SESSION_SECRET` is absent

- Log which OAuth providers are enabled at startup (same style as existing

- `UsersPath: configPath("users.json")`

#### `task/task.go`

Add `OwnerID string` to the `Task` struct.

##### TypeScript SDK refactor (`gen-api-sdk` → `api.gen.ts`)

Inspired by mddb's `createAuthFetch` / `createAPIClient` pattern. The
generated file changes from top-level exported functions to a factory
function that closes over a `FetchFn`.

The generated `api.gen.ts` will have this structure:

```typescript
// Code generated by gen-api-sdk. DO NOT EDIT.
import type { ... } from "./types.gen";

export class APIError extends Error { /* unchanged */ }

export type FetchFn = (url: string, init?: RequestInit) => Promise<Response>;

function makeRequester(fetchFn: FetchFn) {
  return async function request<T>(method: string, path: string, body?: unknown): Promise<T> {
    const init: RequestInit = {
      method,
      headers: { "Content-Type": "application/json" },
      signal: AbortSignal.timeout(60_000),
    };
    if (body !== undefined) init.body = JSON.stringify(body);
    const res = await fetchFn(path, init);
    if (!res.ok) {
      const err = (await res.json()) as ErrorResponse;
      const e = new APIError(res.status, err.error.code, err.details);
      e.message = err.error.message;
      throw e;
    }
    return res.json() as Promise<T>;
  };
}

// createApiClient returns an API client bound to the given fetch function.
// When fetchFn is omitted, globalThis.fetch is used (legacy / unauthenticated mode).
export function createApiClient(fetchFn: FetchFn = globalThis.fetch.bind(globalThis)) {
  const request = makeRequester(fetchFn);
  return {
    getConfig: (): Promise<Config> => request<Config>("GET", "/api/v1/server/config"),
    getMe: (): Promise<UserResp> => request<UserResp>("GET", "/api/v1/auth/me"),
    logout: (): Promise<StatusResp> => request<StatusResp>("POST", "/api/v1/auth/logout"),
    // ... one property per non-SSE route ...

    // SSE routes remain EventSource-based (cookies are sent automatically):
    taskEvents: (id: string, onMessage: (event: EventMessage) => void): EventSource => {
      const es = new EventSource(`/api/v1/tasks/${id}/events`);
      es.addEventListener("message", (e) => {
        onMessage(JSON.parse(e.data) as EventMessage);
      });
      return es;
    },
    // ... other SSE routes ...
  };
}
```

Generator changes in `gen-api-sdk/main.go`:
- Add `FetchFn` type declaration to the static header
- Replace `writeTSJSONFunc` (which emits top-level `export function`) with
  `writeTSJSONMethod` (which emits `methodName: (...) => request(...)` as an
  object property)
- Replace `writeTSSSEFunc` similarly with `writeTSSSEMethod`
- Wrap all methods in the `createApiClient(fetchFn)` factory function body

**Note on SSE + auth**: `EventSource` sends cookies automatically for
same-origin requests. Since the JWT is in a `caic_session` cookie with
`SameSite=Strict`, no changes to the SSE code path are needed for the web
client. Android uses OkHttp (not EventSource), so it sets the `Authorization`
header directly.

After the generator change, run `make types` to regenerate `api.gen.ts`.

##### Kotlin SDK refactor (`gen-api-sdk` → `ApiClient.kt`)

Adapt `writeKotlinClient` in `gen-api-sdk/main.go` to accept a token
provider.

The generated `ApiClient.kt` class signature changes from:
```kotlin

```
to:
```kotlin
class ApiClient(
    baseURL: String,
    private val tokenProvider: (() -> String?)? = null,
)
```

The private `request()` method gains one header injection:
```kotlin
val req = Request.Builder()
    .url(url)
    .method(method, requestBody)
    .header("Content-Type", "application/json")
    .apply { tokenProvider?.invoke()?.let { header("Authorization", "Bearer $it") } }
    .build()
```

The SSE `sseFlow` helper adds the same header to the OkHttp request:
```kotlin
val request = Request.Builder()
    .url("$baseURL$path")
    .header("Accept", "text/event-stream")
    .apply { tokenProvider?.invoke()?.let { header("Authorization", "Bearer $it") } }
    .build()
```

No other changes to the Kotlin generator. Run `make types` to regenerate.

A non-generated wrapper that owns the auth-aware API client:

```typescript
import { createApiClient, FetchFn } from "@sdk/api.gen";

// createAuthFetch wraps a fetch function to inject the caic_session cookie
// (already sent automatically by the browser) and to call onUnauthorized on 401.
// The explicit Authorization header is NOT set here — the cookie handles it.
// This function exists primarily for the onUnauthorized callback and future
// extensibility.
export function createAuthFetch(onUnauthorized: () => void): FetchFn {
  return async (url, init) => {
    const res = await fetch(url, { ...init, credentials: "same-origin" });
    if (res.status === 401) onUnauthorized();
    return res;
  };
}
```

This mirrors the mddb `useApi.ts` pattern but is simpler: because the session
is in a cookie, no token-injection is needed.

#### Auth context (`frontend/src/AuthContext.tsx`, new)

A SolidJS context holding auth state and the API client:

```typescript
import { createContext, useContext, createSignal, onMount, ParentComponent } from "solid-js";
import { createApiClient } from "@sdk/api.gen";
import type { UserResp } from "@sdk/types.gen";
import { createAuthFetch } from "./api";

interface AuthContextValue {
  user: () => UserResp | null;
  ready: () => boolean;
  api: ReturnType<typeof createApiClient>;
  logout: () => Promise<void>;
}

const AuthContext = createContext<AuthContextValue>();

export const AuthProvider: ParentComponent = (props) => {
  const [user, setUser] = createSignal<UserResp | null>(null);
  const [ready, setReady] = createSignal(false);

  const api = createApiClient(createAuthFetch(() => setUser(null)));

  onMount(async () => {
    try {
      const me = await api.getMe();
      setUser(me);
    } catch {
      setUser(null);
    } finally {
      setReady(true);
    }
  });

  const logout = async () => {
    try { await api.logout(); } catch { /* ignore */ }
    setUser(null);
    // Cookie is cleared server-side; page reload is simplest approach.
    window.location.href = "/";
  };

  return (
    <AuthContext.Provider value={{ user, ready, api, logout }}>
      {props.children}
    </AuthContext.Provider>
  );
};

export function useAuth() {
  return useContext(AuthContext)!;
}
```

#### New `android/data/auth/AuthTokenStore.kt`

```kotlin
// Encrypted storage for the caic JWT session token.
package com.fghbuild.caic.data.auth

import android.content.Context
import androidx.security.crypto.EncryptedSharedPreferences
import androidx.security.crypto.MasterKey

class AuthTokenStore(context: Context) {
    private val prefs = EncryptedSharedPreferences.create(
        context,
        "caic_auth",
        MasterKey.Builder(context).setKeyScheme(MasterKey.KeyScheme.AES256_GCM).build(),
        EncryptedSharedPreferences.PrefKeyEncryptionScheme.AES256_SIV,
        EncryptedSharedPreferences.PrefValueEncryptionScheme.AES256_GCM,
    )

    fun getToken(): String? = prefs.getString("token", null)
    fun setToken(token: String) = prefs.edit().putString("token", token).apply()
    fun clearToken() = prefs.edit().remove("token").apply()
}
```

## 6. Security Notes

**Token storage**:
- Web: `HttpOnly` cookie, not accessible to JS; `SameSite=Strict` prevents
  CSRF on all cross-origin requests including `<form>` submits.
- Android: `EncryptedSharedPreferences` backed by Android Keystore.

**OAuth state / CSRF**: HMAC-signed state cookie with 10-minute TTL. Stateless
(no server-side storage). The HMAC uses the same `CAIC_SESSION_SECRET` as JWT
signing — this is safe because the inputs are structurally different.

**Token expiry**: 30-day JWT TTL. No refresh mechanism — users re-authenticate
when the JWT expires. GitLab refresh tokens are stored but not used in v1;
add refresh logic when it becomes a pain point.

**Forge token scope**: the user's OAuth access token has `repo`/`api` scope
and is stored in `users.json`. The file is written with `0o600` permissions
(user-readable only), matching `preferences.json`. Operators should ensure
`~/.config/caic/` is not world-readable.

**No open redirects**: the `return` parameter in the start handler accepts
only `""` (web) and `"app"` (Android). Any other value is rejected with 400.

**Auth bypass in no-auth mode**: when `sessionSecret == nil`, all middleware
is a no-op, all handlers run, all `UserFromContext` calls return `(nil, false)`.
There is no way to accidentally enable partial auth.
